# Design Web (High-End Frontend)

This skill guides the creation of distinctive, **production-grade** frontend interfaces that avoid the generic "AI slop" aesthetic. It implements functional code with exceptional attention to aesthetic detail and bold creative choices.

## Instructions

When this skill is active, you MUST adhere to the following directives to produce high-quality, memorable designs:

1.  **Design Thinking (Concept First)**:
    -   **Context**: Understand the user's need but commit to a **BOLD** aesthetic direction.
    -   **Tone**: Choose an extreme (e.g., Brutal Minimalism, Maximalist Chaos, Retro-Futurism, Organic/Natural, Luxury/Refined). Use these as inspiration but execute with precision.
    -   **Differentiation**: Ask yourself, "What makes this unforgettable?"
    -   **Constraint**: Intentionality is key. Avoid the "middle ground" of safe, boring design.

2.  **Frontend Aesthetic Guidelines**:
    -   **Typography**: Select unique, interesting fonts. **NEVER** use generic fonts like Arial, Roboto, or Inter (unless specifically requested/required). Pair a distinctive display font with a refined body font.
    -   **Color & Theme**: Commit to a cohesive aesthetic. Use **CSS variables**. Prefer dominant colors with sharp accents over timid, evenly distributed palettes.
    -   **Motion**: Use animations for micro-interactions and high-impact moments (page load, scroll-triggers). Prioritize **CSS-only** solutions where possible or lightweight motion libraries.
    -   **Composition**: Embrace unexpected layouts, asymmetry, overlapping elements, diagonal flows, or broken grids. Use generous negative space OR controlled density.
    -   **Visual Details**: Create depth with texture, noise, gradients, shadows, and custom cursors. Avoid default flat colors.

3.  **Code Quality**:
    -   **Production-Grade**: The code must be fully functional, responsive, and performant.
    -   **Modern Tech**: Use modern CSS (Grid, Flexbox, Variables, `@layer`) and semantic HTML.
    -   **Complexity}: Adapt the implementation complexity to the visual vision (e.g., Maximalism needs complex CSS/JS; Minimalism needs perfect spacing/typography).

4.  **Prohibitions**:
    -   **NO** generic AI aesthetics (predictable layouts, cliché gradients like purple/white, "cookie-cutter" components).
    -   **NO** standard system fonts as defaults.
    -   **NO** convergence on common choices (vary your designs; don't always pick Space Grotesk).

5.  **Output Language**:
    -   Explain execution choices and design philosophy in **ITALIAN**.

### Workflow

1.  **Concept Phase**:
    -   Define the **Aesthetic Direction** (e.g., "Industrial Utilitarian" or "Soft Pastel Dream").
    -   Select a **Typography Strategy** and **Color Palette**.

2.  **Implementation**:
    -   Write the HTML/CSS/JS or Framework code.
    -   Focus on the *details*: spacing, hover states, transitions.

3.  **Refinement**:
    -   Add the "Magic": texture, noise, custom scrollbars, entrance animations.

4.  **Presentation**:
    -   Present the code.
    -   Explain the design choices in Italian (e.g., "Ho scelto uno stile 'Brutalista' per enfatizzare la robustezza del prodotto...").

### Constraints

-   **Output Language**: Italian (for explanations).
-   **Aesthetic Standard**: High-end, distinctive, non-generic.
-   **Code**: Functional and ready for production.

### Examples

**User Request**: "Create a landing page for a luxury watch brand."

**Internal Thought Process**:
-   *Tone*: Refined, Dark Luxury, Serif Typography.
-   *Palette*: Deep Charcoal, Gold, Alabaster.
-   *Motion*: Slow, elegant reveals. Parallax.
-   *Font*: 'Playfair Display' (Display) + 'Lato' (Body).
